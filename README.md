# English Manual

ChatOps is a lightweight, dependency-free, and sleek Telegram operations automation control system developed in Go. It allows developers and sysadmins to deploy code, monitor servers, configure Cloudflare DNS, and manage SSH node additions directly through a unified Telegram interface with a zero-footprint UI.

It consists of two main components:
- **Master (Control Plane)**: Receives Telegram Bot commands, schedules FIFO queues, manages Cloudflare/DNS, issues cross-compiled binaries, and securely dispatches deployment tasks over encrypted gRPC.
- **Agent (Worker Daemon)**: Long-running worker daemon that communicates with the Master, reports local system stats, builds Docker containers (via Fallback/Railpack builder), and updates Caddy routing dynamically.

---

## 🚀 Key Features

* **Sleek & Clean Chatbot UI**: Uses a custom "in-place bubble editing and physical cleanup" mechanism. Interactive steps update the same message bubble, and user inputs (passwords, git URLs) are physically deleted within milliseconds to keep the chat history completely clean.
* **gRPC TLS Encryption & In-Memory Certificates**: Supports secure gRPC tunnels. If no physical TLS certificates are provided, the Master automatically generates a secure 10-year ECDSA self-signed certificate in memory, and the Agent skips verification for seamless TLS-encrypted communication without exposing private keys.
* **Auto-SSH Remote Node Deployment**: Allows adding remote servers directly through Telegram. The Master cross-compiles the Agent binary for `linux-amd64`, uploads it securely via SSH stdin (avoiding SFTP requirements), configures execution permissions, starts it as a daemon, and monitors the gRPC handshake registration in 15 seconds.
* **GitHub Releases Hot Update**: Features a one-click update engine. The Master queries the GitHub Releases API, performs semver comparison, downloads the target binary, and swaps itself gracefully. Agent upgrades are dispatched via gRPC; the Agent dynamically downloads the matching OS/Arch binary and detaches to restart.
* **Serialized FIFO Queues & Overload Protection**: Implements a per-node FIFO task queue to serialise build tasks. Includes an interceptor that halts queue execution if the Agent's CPU exceeds 85% or Memory exceeds 90%, automatically waking up once metrics cool down.
* **Auto DNS & Proxy Routing**: Integrates with Cloudflare REST APIs to provision CNAME/A records. Directly controls Agent-side Caddy servers to configure secure reverse proxy configurations with automatic Let's Encrypt / ZeroSSL certificate activation.

## 🛠️ Getting Started

### 1. Copy Configuration Templates
Copy the template files and fill in your credentials:
```bash
# Copy Master configurations
cp master.yaml.example master.yaml

# Copy Agent configurations
cp agent.yaml.example agent.yaml
```
* **master.yaml**: Insert your Telegram Bot Token.
* **agent.yaml**: Insert the Master address and the `communication_token` generated by the Master.

### 2. Compile and Run

#### Running the Master node:
```bash
# Compile Master binary
go build -o build/gopass-master.exe ./cmd/gopass-master

# Start Master node
./build/gopass-master.exe -config master.yaml
```
*Note: The first Telegram user to send any message to the Bot will be automatically bound as the supreme Master administrator.*

#### Running the Agent node:
```bash
# Compile Agent binary
go build -o build/gopass-agent.exe ./cmd/gopass-agent

# Start Agent node
./build/gopass-agent.exe -config agent.yaml
```

   ```bash
   go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
   go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest
   ```
2. **Run Protoc compiler** (Windows example using local `protoc-bin`):
   ```powershell
   $env:PATH = "C:\Users\shixu\go\bin;" + $env:PATH
   .\protoc-bin\bin\protoc.exe --go_out=. --go_opt=paths=source_relative --go-grpc_out=. --go-grpc_opt=paths=source_relative pkg/proto/protocol.proto
   ```

# 中文使用手册

ChatOps 是一个基于 Go 语言开发，轻量、免外部数据库依赖的极简无痕 Telegram 运维自动化控制系统。它支持开发人员和系统管理员直接通过 Telegram 聊天界面进行应用构建部署、服务器监控、Cloudflare DNS 解析配置，以及一键 SSH 部署并添加新节点。

该项目包含两个核心组件：
- **Master (控制端)**：接收 Telegram 机器人的交互指令、维护基于 FIFO 的任务队列、管理 Cloudflare DNS 绑定、提供二进制交叉编译分发，并通过安全的 gRPC 长连接信道向被控端下发部署和管理指令。
- **Agent (被控端)**：长效常驻后台的被控节点守护程序。它主动回连 Master，上报系统 CPU/内存/磁盘负载指标，调用容器编译器（支持 Fallback/Railpack 引擎）构建部署 Docker 容器，并刷新本地 Caddy 代理路由。

---

## 🚀 核心特性

* **无痕流畅交互**：引入了“原地气泡编辑与历史物理清除”设计。所有的向导步骤都在同一个消息气泡中更新，用户发送的敏感凭证（如 SSH 密码、私钥、Git 仓库地址）在系统接收后会**以毫秒级速度物理删除**，保障聊天面板绝无痕迹。
* **gRPC TLS 安全通信与内存自签名证书**：支持 gRPC 链路 TLS 加密。若用户未配置物理证书文件，Master 会在启动时自动在内存中动态生成有效期为 10 年的 ECDSA 自签名证书，Agent 配合 `SkipVerify` 选项即可建立 TLS 加密通信，杜绝证书明文写盘和泄漏。
* **一键 SSH 远程部署被控节点**：支持在 Telegram 界面直接输入目标机器 IP、端口、用户名和密码/私钥。Master 会在本地交叉编译生成适合 Linux amd64 架构的 Agent 运行包，通过 SSH 标准输入流免 SFTP 写入远程机器，自动修改执行权限并后台静默启动守护进程，在 15 秒内自动完成 TLS 握手注册。
* **GitHub Releases 一键热更新**：支持控制台一键检查升级。Master 可请求 GitHub API 进行语义化版本比对，下载匹配资产并执行进程热覆盖替换；同时支持通过 gRPC 向指定 Agent 节点下发升级任务，由 Agent 自适应 runtime 解析系统架构（解决 OS/Arch 差异）完成在线热更新与自动重连。
* **串行 FIFO 队列与防爆载熔断器**：为每个被控节点维护专属 FIFO 任务序列，防止多任务并发构建抢占资源。若 Agent 节点的 CPU 超过 85% 或内存超过 90%，拦截器会自动挂起队首构建任务，待节点指标冷却后自动唤醒分发。
* **自动反代与 DNS 解析绑定**：深度对接 Cloudflare 接口自动配置 A 或 CNAME 解析，并支持 CDN 代理模式。Agent 侧自动控制 Caddy 反向代理，支持非 localhost 域名的自动 https (Let's Encrypt / ZeroSSL) 证书生成与续签。

## 🛠️ 快速上手

### 1. 配置准备
从模板复制配置文件：
```bash
# 复制控制端配置模板
cp master.yaml.example master.yaml

# 复制被控端配置模板
cp agent.yaml.example agent.yaml
```
* **master.yaml**: 填入您的 Telegram Bot Token。
* **agent.yaml**: 填入 Master 端的监听地址，并配置 Master 启动时控制台输出的 `communication_token`（安全通信密钥）。

### 2. 编译并运行

#### 启动 Master 控制端：
```bash
# 编译 Master 端二进制
go build -o build/gopass-master.exe ./cmd/gopass-master

# 运行 Master 端
./build/gopass-master.exe -config master.yaml
```
*注：首次启动 Master 时，首位向 Telegram Bot 发送任意指令的用户将自动绑定为系统最高管理员。*

#### 启动 Agent 被控端：
```bash
# 编译 Agent 端二进制
go build -o build/gopass-agent.exe ./cmd/gopass-agent

# 运行 Agent 端
./build/gopass-agent.exe -config agent.yaml
```
